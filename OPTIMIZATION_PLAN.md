## 三、技术方案（四项优化）

### 1. CFD/FEM温度求解作为验证标准 [P1]

#### 问题
当前引擎使用半经验模型。关键缺陷：
- 未求解柱坐标下的完整热传导方程（缺少 `(1/r)∂T/∂r` 径向项）
- 边界条件用经验系数模拟，非精确 Dirichlet/Neumann
- 无验证手段衡量模拟精度

#### 方案

**后端严格FDM求解器（Python）**

```
文件: server/solver.py
依赖: numpy, scipy, fastapi, uvicorn
```

核心方程（柱坐标2D轴对称）：
```
∂T/∂t = α · (∂²T/∂r² + (1/r)·∂T/∂r + ∂²T/∂h²)
```

边界条件：
- 壁面：`-k·∂T/∂r|_wall = h_wall · (T_wall - T_ambient)`（第三类边界）
- 液面：`-k·∂T/∂h|_surface = h_surf · (T_surface - T_ambient)`（蒸发散热）
- 中心轴：`∂T/∂r|_r=0 = 0`（轴对称）

**前端UI集成**
- 右上角新增精度指标徽章：`模拟精度: ±X°C (vs CFD)`
- 每次求解完成后自动/手动发起验证请求
- 偏差曲线叠加在温度-时间图上（虚线表示CFD参考值）

**技术决策：为什么不直接用FEM？**
FDM在结构网格上精度足够且速度快，与当前数学形式一致。FEM留给更复杂的非规则几何（目标2）。

### 2. 图纸上传→网格→3D展示→空间温度场 [P2]

#### 分阶段实施

**Phase 1：浏览器端2D截面绘制（纯前端，可独立交付）**

```
新增文件: mesh-editor.js
```

工作流：
1. 用户通过 Canvas 绘制截面轮廓（鼠标绘制 或 拖入参考图描边）
2. 提取轮廓点集 → 旋转生成轴对称3D几何
3. 自动生成结构化网格（20×24或自适应密度）
4. 在自定义几何上运行FDM求解
5. Three.js 渲染几何 + 温度彩色映射

核心交互：
```
┌──────────────────────┐
│  绘图模式 / 导入模式   │
│  ┌──────────────────┐│
│  │   Canvas 绘图区    ││
│  │   左键: 加点       ││
│  │   右键: 闭合       ││
│  │   拖拽: 调整轮廓    ││
│  └──────────────────┘│
│  [重置] [撤销] [生成] │
└──────────────────────┘
```

**Phase 2：STL导入+后端网格划分（需后端）**
- 后端用 meshio + GMSH API 自动划分非结构网格
- scikit-fem 或 FEniCS 求解
- 结果回传前端

**交互限制说明**
| 功能 | 可行性 | 实现方式 |
|------|--------|----------|
| 2D截面绘制→轴对称3D | ✅ 纯前端 | Canvas + Three.js LatheGeometry |
| STL解析渲染 | ✅ Three.js STLLoader | 已有成熟库 |
| 非结构网格划分 | ❌ 前端不可行 | 后端 GMSH API |
| 3D FEM求解 | ❌ 前端不可行 | 后端 scikit-fem |

### 3. 质感优化 [P1]

#### 3.1 玻璃拟态面板
```css
.glass-panel {
    background: rgba(255,255,255,0.72);
    backdrop-filter: blur(20px);
    -webkit-backdrop-filter: blur(20px);
    border: 1px solid rgba(255,255,255,0.3);
    box-shadow: 0 8px 32px rgba(0,0,0,0.08);
}
```
应用范围：左侧参数面板、右侧图表面板、状态栏

#### 3.2 3D场景加强
| 改进项 | 实现方式 |
|--------|----------|
| HDR环境贴图 | `PMREMGenerator` + EXR环境图 |
| 液面反射 | `MeshPhysicalMaterial` 设置 `transmission`, `roughness`, `metalness` |
| 柔和阴影 | `renderer.shadowMap` + 聚光灯阴影贴图 |
| 地面反射 | 半透明反射平面 |

#### 3.3 颜色映射升级
当前：分段线性映射（红→橙→黄→绿→蓝）
替换：perceptually uniform colormap（Inferno / Viridis）

```javascript
// 引入 color-map.js 依赖
const cmap = getColormap('inferno', 256);
const color = cmap[normalizedTemp * 255];
```

Inferno 在工业/科学模拟中视觉专业度最高，且对色觉障碍友好。

#### 3.4 微交互
- 滑块追踪值显示 + 光晕跟随
- 按钮点击 ripple 效果
- 探针放置时的飞线动画（从探针面板到3D位置）
- 参数变化时 liquid mesh 做 morph 过渡（非立即重建）
- 时间轴拖拽时温度场实时渐变

#### 3.5 粒子系统（蒸发效果）
使用 Three.js Points，在液面位置随机生成上升粒子：
- 粒子大小随高度衰减
- 透明度随高度渐变消失
- 粒子密度与 `T_liquid - T_ambient` 正相关

## 六、关键技术决策

| 决策 | 选项 | 结论 | 理由 |
|------|------|------|------|
| 后端框架 | Flask vs FastAPI | **FastAPI** | 异步+自动文档，适合科学计算API |
| PINN框架 | PyTorch vs TensorFlow | **PyTorch** | 研究生态更广，调试方便 |
| 颜色映射 | Inferno vs Viridis | **Inferno** | 工业视觉更专业，低亮度区对比度更好 |
| 3D渲染加强 | 纯Three.js增改 vs 换库 | **增改Three.js** | 已有完整实现，换库成本高 |
| 网格划分 | 前端做 vs 后端做 | **前端(简单)/后端(复杂)** | 分阶段策略，前端做轴对称足够 |
| 状态管理 | 全局对象 vs 框架 | **全局对象** | 项目规模小，引入框架增加复杂度 |

                  │  · MaxAbsError │
                  │  · R² score    │
                  │  · 边界层偏差  │
                  └────────────────┘
```

### 10.2 评估指标

| 指标 | 公式 | 说明 |
|------|------|------|
| RMSE | √(Σ(T_pred-T_ref)²/N) | 整体均方根误差 |
| MaxAE | max\|T_pred-T_ref\| | 最大绝对偏差 |
| R² | 1 - Σ(T_pred-T_ref)²/Σ(T_ref-μ)² | 拟合优度 |
| 边界层偏差 | 杯壁/液面附近1-3层的平均偏差 | 关键区域专项评估 |
| 全局偏差中位数 | median(\|T_pred-T_ref\|) | 抗离群值评估 |

### 10.3 自动校准流程

```
每次CFD求解完成后 → 自动启动精度评估
     ↓
若 RMSE > 阈值(默认2°C)：
    → 前端显示黄色预警
    → 建议用户切换到精确模式
    → 或触发PINN增量训练
     ↓
若 RMSE ≤ 阈值：
    → 显示绿色"高置信度"状态
    → 更新HUD精度指标
```

## 十二、可扩展性架构：多物质/多物理场 [NEW]

### 12.1 当前限制

- 仅支持单相流体（液体内无固体/气态）
- 仅支持热传导，无对流/相变/化学反应
- 几何形状固定为轴对称杯体
- 损失函数编码在求解器内部，不可配置

### 12.2 目标扩展场景

| 场景 | 物理特征 | 新增需求 |
|------|----------|----------|
| **搅拌** | 强制对流 + 湍流 | 流体速度场 (Navier-Stokes)、混合模型 |
| **煮饭** | 固液耦合 + 相变(水沸腾) | 两相流、汽液相变模型、多孔介质 |
| **多液体层** | 液体分层 + 界面传热 | 多物质输运方程、界面追踪 |
| **炖煮/蒸煮** | 相变 + 化学反应 + 质量传递 | 蒸发冷凝模型、Arrhenius反应动力学 |

### 12.4 损失函数插件定义规范

    name: str                    # 唯一标识符
    physics_fields: list[str]    # 依赖物理场 (T, v, p, C, H...)
    config_schema: dict          # 配置参数JSON Schema
    
    def compute(self, model, inputs, outputs) -> tf.Tensor:
        """计算该损失项"""
        raise NotImplementedError

    def validate_config(self, config: dict) -> bool:
        """验证配置合法性"""
        return True
```

```python
# plugins/convection.py — 对流插件示例
class ConvectionLoss(LossPlugin):
    name = "convection"
    physics_fields = ["T", "v_x", "v_y"]
    
    def compute(self, model, inputs, outputs):
        # ∂T/∂t + v·∇T - α∇²T = 0
        with tf.GradientTape(persistent=True) as tape:
            T = outputs["T"]
            v_x, v_y = outputs["v_x"], outputs["v_y"]
            T_t = tape.gradient(T, inputs["t"])
            T_x = tape.gradient(T, inputs["x"])
            T_y = tape.gradient(T, inputs["y"])
        return tf.reduce_mean((T_t + v_x*T_x + v_y*T_y - alpha*laplacian(T))**2)
```
